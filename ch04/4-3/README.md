# 실습과제 1
### 메모리 누수를 설명하라.
동적으로 할당한 메모리를 해제하지 않거나, 해당 메모리를 가리키던 포인터를 다른 주소로 변경하여 원래의 메모리에 더 이상 접근할 수 없게 되는 경우, 또는 반복문이나 무한루프에서 메모리를 계속 할당만 하고 해제하지 않는 경우 등으로 인해 사용이 끝난 메모리가 반환되지 못하는 것이다. 
### 메모리 누수가 왜 심각한 문제를 발생시키는지 설명하라.
메모리 누수가 발생하면 사용하지 않는 메모리가 계속 점유되어 가용 메모리가 감소하고 그로 인해 프로그램의 실행 속도가 느려지거나 성능이 저하되는 문제가 발생하한다 또한 메모리가 부족해지면서 프로그램이 비정상적으로 종료되거나 시스템 전체의 불안정성을 초래할 수 있다.
### 메모리 누수를 방지하는 방법을 설명하라.
메모리 누수를 방지하기 위해서는 동적으로 할당한 메모리를 사용이 끝난 즉시 반드시 해제하는 것이 가장 기본적이다.
또한 C++에서는 스마트 포인터(unique_ptr, shared_ptr)를 사용하여 메모리가 자동으로 관리되게 하거나 Valgrind와 같은 메모리 검사 도구를 활용하여 누수를 사전에 발견하고 수정하는 방법도 있다. 
### C++ 스마트 포인터 클래스에 대해 조사하라
스마트 포인터는 C++11이상 표준에서 제공되는데 동적할당 메모리를 자동으로 해제해준다. std네임스페이스에 정의되어있으며 3가지가 있다.
| 스마트 포인터       | 특징                                   | 사용 예시                                      | 장점                          | 주의점                          |
|----|----|----|----|----|
| 유니크포인터(unique_ptr) | 단일 소유권, 복사 불가, 이동만 가능    | `std::unique_ptr<int> p = std::make_unique<int>(10);` | 메모리 누수 방지, 성능 우수   | 복사 불가, 소유권 이전 시 `std::move` 필요 |
| 쉐어드포인터(shared_ptr) | 참조 카운팅 기반 다중 소유권           | `std::shared_ptr<int> p = std::make_shared<int>(10);` | 여러 객체가 같은 리소스 공유 가능 | 순환 참조 발생 시 메모리 해제 불가 |
| 약포인터(weak_ptr)   | `shared_ptr`의 보조 포인터, 소유권 없음 | `std::weak_ptr<int> wp = sp;`                  | 순환 참조 문제 해결            | 사용 전 `lock()`으로 유효성 확인 필요 |
