# 실습과제 1
### 접근지정자가 왜 필요한가?
접근지정자는 클래스의 멤버에 대한 외부 접근을 제한하여 객체의 데이터를 보호하기 위해 필요하다. 이를 통해 중요한 멤버변수는 외부에서 직접 접근하지 못하도록 하고, 허용된 멤버함수를 통해서만 접근하게 하여 잘못된 값이 저장되는 것을 방지할 수 있다.

### 생성자와 setter 멤버함수는 모두 멤버변수를 설정하는 기능이다. 차이점을 설명하라.
생성자와 setter 함수는 모두 멤버변수를 설정하는 기능을 가지지만, 생성자는 객체가 생성될 때 한 번만 호출되어 초기값을 설정하는 데 사용되고, setter 함수는 객체가 생성된 이후에도 필요에 따라 값을 변경하기 위해 사용된다.

### 객체지향형 언어의 특징인 캡슐화와 정보은닉을 설명하라.
캡슐화는 데이터와 함수를 하나의 클래스안에 모아 놓고 외부로부터 안전하게 보호하는 기능이다.
정보은닉은 멤버변수를 private으로 선언하고 해당 변수를 접근하는 함수를 별도로 정의해서 안전한 형태로 멤버 변수에 접근할 수 있도록 하는 기능으로 컴파일 과정에서 쉽게 오류를 발견 할 수있다.
