# 실습과제 1
### 함수를 사용하는 이유를 설명하라.
함수는 코드의 중복을 방지하고 재사용성을 높이기 위해 사용된다. 또한 프로그램을 기능별로 나누어 구조를 명확하게 하고, 가독성과 유지보수성을 향상시킨다.

### 짧은 코드를 갖는 함수를 자주 호출하면 발생하는 함수의 오버헤드를 설명하라
함수 오버헤드란 함수 호출 시 실제 코드 실행 외에 발생하는 추가적인 처리 비용을 의미한다. 이는 리턴 주소 저장, 매개변수 전달 및 초기화, 함수 종료 후 복귀 과정 등을 포함한다. 특히 코드 길이가 짧은 함수를 자주 호출할 경우, 실제 연산보다 이러한 오버헤드가 차지하는 비율이 커져 전체 실행 시간이 증가하고 성능이 저하될 수 있다.

### 12페이지 예제에서 인라인함수가 치환되면 private 멤버 변수를 직접 접근하게 된다. 문법오류가 아닌가?
코드를 그대로 쓰면 오류지만, 클래스의 private, public은 컴파일 단계에서만 의미를 갖고, 기계어 수준에서는 단순한 데이터와 함수로 변환된다. 따라서 인라인 함수 치환 시 private 멤버에 접근하는 것처럼 보여도 문제되지 않는다.

### 인라인함수는 소스코드에서는 함수이지만 기계어로 번역되면 함수가 아니다. 이렇게 복잡하게 만든 이유를 설명하라. 다시 말해, 처음부터 함수를 안 쓰면 안되나요?
함수로만 사용하게 되면 호출 시 오버헤드가 발생하여 시간과 메모리를 더 사용하게 되므로 한 줄 정도의 짧은 함수는 인라인 함수로 처리하여 실행 시간과 메모리 사용을 줄일 수 있다.
